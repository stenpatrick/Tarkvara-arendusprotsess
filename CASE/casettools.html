CASE

What are CASE Tools?

Computer-Aided Software Engineering (CASE) tools are tools that provide automated assistance in software development. The adoption of CASE tools aims to reduce software development time and costs, and improve the quality of developed systems.

Why Use Them?

CASE tools enable the development of systems that are easier to test and maintain, and that include high-quality documentation. In short, software developed with CASE tools has improved quality because there are fewer defects.

What are the Potential Drawbacks of CASE Tools?

  • The main problem with CASE tools is usually lack of interoperability.
  • Also, it's often necessary to learn how to use a particular CASE tool, depending on its complexity.
  • Some CASE tools require additional payments, especially SaaS providers.

How are CASE Tools Classified?

CASE tools can be classified in various ways based on their specific purposes. CASE tools that support the system development life cycle are classified into two analyses, for example:

  • "Upper-level" CASE tools support analysis and design. They are primarily used for user analysis and documentation. They are mainly intended for visualization, creating different diagrams, and generating documentation. They support traditional diagram languages (entity-relationship diagrams), data models, UML diagrams, etc.
  • "Lower-level" CASE tools focus on implementation, where models become actual software products. They support generating database structures, generating code, conducting tests, version control, configuration management, reverse engineering, etc.
